## Conventions
- [CARD-001] GitHub Actions workflows should declare an explicit top-level `permissions:` block (least privilege, e.g. `contents: read`) rather than inheriting the repo-default `GITHUB_TOKEN` scope; add per-job write scopes only where a push/publish step needs them (the deferred registry-push job in `ci.yml` will).
- [CARD-001] Card scopes for this project are designed to map 1:1 onto the slices in `docs/implementation/06-vertical-slices.md`; card-slicer should treat that doc as strong evidence for the right-sizing verdict on infra/foundation cards, since the project's own architects already vertical-sliced the build order there.

- [CARD-001] Config is parsed by `config.Load(getenv func(string) string) (*Config, error)`, not by reading `os.Getenv` inside the package. `main` injects `os.Getenv`; tests inject a `map`-backed getenv. Keeps config parsing a pure, table-testable function with no process-env mutation.
- [CARD-001] Frontend assets are embedded through a single `web` package (`web/embed.go` with `//go:embed`), imported as `nyx-drop/web`. Go embed cannot reach parent directories, so the directive must live in the `web/` tree, not in `internal/server`. Later cards add files to the embed set here.
- [CARD-001] `server.New(server.Deps{...}) (http.Handler, error)` takes a `Deps` struct, not a positional arg list, so later cards add wiring (store, locks, auth) by adding fields without breaking the signature. Health readiness is an injected `func(context.Context) error` (S1 stub returns nil; CARD-002 supplies the real DB-ping + data-dir-writability check).

- [CARD-002] (designed; ADR-0003, in flight on PR #10) `internal/store` stores timestamps as fixed-width RFC 3339 with always-9-digit fractional seconds in UTC (format const `"2006-01-02T15:04:05.000000000Z07:00"`, value normalized to UTC first). Fixed width is required: plain `time.RFC3339` (second precision) loses sub-second data so the round-trip `time.Equal` check fails on a nanosecond clock value, and `time.RFC3339Nano` trims trailing zeros to variable length which breaks lexicographic `ORDER BY` / `expires_at <= ?` comparisons (e.g. `"05.1Z"` sorts BEFORE `"05Z"` yet is chronologically later). The fixed 9-digit form round-trips nanoseconds exactly AND sorts lexicographically = chronologically. Parse back with `time.Parse(time.RFC3339Nano, s)` (parsing is length-agnostic). This is the storage encoding only; API wire serialization (CARD-003) is a separate choice.
- [CARD-002] (designed; in flight on PR #10) The store is Clock-free by design: every business time enters as an explicit parameter (`Create`'s `Site` fields, `Touch`'s `updatedAt`/`expiresAt`, `SetPermanent`'s `newExpiry`, `Expired`'s `now`). Invariant 9 (never `time.Now()` in store/handlers) is satisfied structurally â€” the caller supplies `clock.Now()`; the store never imports `clock`. Keep it that way; do not add a `Clock` to `store`.
- [CARD-008] (designed; ADR-0006, in flight on PR #11) The OAuth `state` cookie MUST be `SameSite=Lax`, never `Strict`: the callback from github.com is a cross-site top-level GET navigation, and a `Strict` cookie would not accompany it, breaking every sign-in. `__Host-oauth-state` (or `oauth-state` under `SCHEME=http`) therefore carries `SameSite=Lax` + `HttpOnly` + `Secure` + `Path=/` + no `Domain`, 10-min `Max-Age`; the session cookie is likewise `Lax`.
- [CARD-008] (designed; in flight on PR #11) `internal/auth` is the pure core-logic layer for auth: HMAC token sign/verify, session/state claim verification (`now` injected, never `time.Now()`), and the `OriginAllowed`/`AuthorizedUser` predicates are all framework-free and unit-tested to the â‰Ą90% layer aggregate. The only I/O adapter is `OAuth.ExchangeLogin` (token exchange + `/user` fetch) with injectable base URLs + `*http.Client` for the stub GitHub. Cookie construction (`*http.Cookie`, scheme-dependent) lives at the http edge in `internal/server/cookies.go`, not in `auth`.
- [CARD-008] (designed; in flight on PR #11) CARD-008 registers the `/api/admin/` subtree on `apexMux` wrapped in `requireSession(api-mode)` (401 JSON when unauthenticated) as a placeholder; **CARD-009 must register only MORE-SPECIFIC patterns** (e.g. `/api/admin/sites`) under the same guard and must NOT re-register `/api/admin/` (`ServeMux` panics on duplicate patterns). The stub `/admin` page route (200 when authed, 302â†’`/auth/login` otherwise) is likewise replaced-in-place by CARD-009's real `admin.html`.
- [CARD-012] Project doctrine that **binds phase agents** lives in `PROTOCOL-ADDENDUM.md` (read after the plugin's `AGENT-PROTOCOL.md` on every dispatch), not in `KNOWLEDGE.md` â€” this file is captured, orchestrator-owned notes, not the authoritative contract. When a KNOWLEDGE gotcha is promoted to doctrine, the doctrine text becomes authoritative and the KNOWLEDGE entry should shrink to a pointer. Phase agents must never edit `KNOWLEDGE.md`; only `/kanban` writes it.

- [CARD-001] Site-host detection (`siteLabel`) is host-confusion-safe by construction: it matches on the full-dot suffix `"." + baseDomain` (never a bare `HasSuffix` on `baseDomain`), then rejects an empty label and any label still containing a dot. This defeats the classic suffix-confusion vectors (`evilsites.nyxhub.net`, `xsites.nyxhub.net`, `sites.nyxhub.net.evil.com`, `.sites.nyxhub.net`) â€” all resolve to a branded 404, none are treated as a site. CARD-003 must preserve this exact shape when it replaces the site-host stub with real serving, and must decide trailing-dot (FQDN) handling explicitly, since `normalizeHost` does not strip it (today it fails safe to 404).

## Gotchas
- [CARD-001] Invariant 9 (handlers use the injected Clock, never `time.Now()`) governs BUSINESS time (expiry). Request-latency measurement in the logging/metrics middleware must use `time.Now()`/`time.Since` (the monotonic wall clock) â€” a frozen fake Clock would report `dur_ms=0` and is semantically wrong for latency. Do not route latency timing through `clock.Clock`.
- [CARD-001] A single-pass "strip trailing `:port` if candidate is non-empty and colon-free" rule is idempotent on its own output, but composing it behind a bracket-unwrap step is NOT automatically idempotent â€” bracket content (e.g. `0:0` from `[0:0]`) can itself match the bare `host:port` shape and get re-stripped on a second call. Fix: route bracket content through the exact same idempotent single-pass helper before returning, rather than returning it raw. Any future host/address normalizer with an idempotency invariant should route every extraction branch through one shared, provably-idempotent primitive rather than duplicating similar-but-not-identical logic per branch â€” and back the invariant with live fuzzing (`go test -fuzz`, 30-60s), not just the seed corpus, before trusting it.
- [CARD-001] `.gitignore` entries for build artifacts that share a name with a real source directory must be anchored to repo root (e.g. `/drop`, not `drop`) â€” an unanchored pattern matches any path component with that name anywhere in the tree (here, `cmd/drop/` silently vanished from `git status`/`git add`). Worth a quick `git ls-files` sanity check after any `.gitignore` edit that adds a short/common name.

- [CARD-001] The `coverage_target` ("90% on the core logic layer") is an **aggregate over the layer's statements**, not a per-function floor. Measure it as `go test -coverprofile=â€¦ ./...` then sum covered/total statements across the named packages and functions â€” do not fail an individual function that sits below the number. A per-function reading blocked CARD-001 on `renderNotFound` at 71.4% while the layer stood at 92.0%, and the only way to satisfy it would have been to inject a breakable `fs.FS` into a compile-time `embed.FS` parse â€” deviating from a merged design's fixed signature to move a metric. Unreachable error paths are not coverage debt. CARD-012 lifts this rule into `PROTOCOL-ADDENDUM.md`.

- [CARD-001] `/healthz` and `/metrics` are matched in `top()` before host routing (invariant 8), so on a wildcard ingress **every site subdomain will serve unauthenticated Prometheus runtime metrics on any Host** unless the ingress excludes those paths. Not a defect â€” the spec mandates host-independent `/metrics`, and the exposition carries no secrets â€” but **CARD-011's Helm/ingress work must decide** whether `/metrics` is reachable from the public internet or restricted to in-cluster scraping / a separate port.
- [CARD-001] `internal/config/size.go`'s `ParseSize` accepts bare single-letter `K`/`M`/`G` decimal aliases (see `sizeSuffixes`) in addition to `KB/MB/GB` and `KiB/MiB/GiB`. The suffix set is stated in three places â€” the unexported `sizeSuffixes` var, `ParseSize`'s doc comment, and `README.md`'s config table. They drifted once (both docs omitted the bare aliases; fixed in `1f9eef2`); keep all three in sync when ADR-0001's suffix set changes.
- [CARD-001] A multiply-overflow guard of the form `n > math.MaxInt64/mult` cannot be exercised by an input whose *digits alone* exceed int64 â€” `strconv.ParseInt` rejects those first. To test such a guard you need a value that fits int64 on its own but overflows after the suffix multiply (e.g. `9223372036854776kb`). A no-panic fuzz target will not catch a broken guard either. Check guard coverage with `go tool cover -func`, not by eyeballing the table.
- [CARD-001] `pr-review` lens agents running in parallel can collide on the `gh` review-POST round trip: one agent's review payload was silently submitted under another concurrent lens's content (detected by checksumming the request body and independently re-fetching the created review by id). The `[simplicity]` review landed twice on PR #3 as a result. Mitigation: write the review JSON to a lens-plus-random-suffix-named file, verify its checksum immediately before and after posting, and re-fetch the created review by id before trusting the POST response. Future `pr-expert-reviewer` dispatches should build this verify-after-post step in.
- [CARD-001] Go's `http.Server.IdleTimeout` falls back to `ReadTimeout` (**not** `ReadHeaderTimeout`) when zero, so setting only `ReadHeaderTimeout` leaves idle keep-alive connections unreaped â€” a resource-exhaustion vector. Always set `IdleTimeout` explicitly. Be cautious adding `WriteTimeout`/`ReadTimeout` to the process `http.Server` before the upload/site-serving cards land: a fixed deadline sized without knowing `MAX_UPLOAD_SIZE`/`MAX_SITE_SIZE` risks truncating legitimate large requests and responses. **CARD-003/004 must revisit `WriteTimeout`** once those limits are wired into the server.

- [CARD-002] (designed; in flight on PR #10) `modernc.org/sqlite` specifics designed around: (1) open with `sql.Open("sqlite", path+"?_pragma=...")` and `db.SetMaxOpenConns(1)` â€” WAL is a persisted DB property, `busy_timeout`/`foreign_keys` are per-connection, so with a single pooled conn all four pragmas hold; (2) WAL is a no-op for `:memory:`, so store tests MUST use a real file under `t.TempDir()`, never `:memory:`; (3) `sql.Open` does not connect â€” `Open()` must `PingContext` to force connection and surface DSN/pragma errors at startup; (4) `PRAGMA user_version = N` cannot take a bound parameter â€” build the statement with the trusted int via `fmt.Sprintf`, never a user value; (5) map a primary-key collision by `errors.As` to `*sqlite.Error` and comparing `.Code()` to `sqlitelib.SQLITE_CONSTRAINT_PRIMARYKEY` (import `modernc.org/sqlite/lib`), not by string-matching the message.
- [CARD-002] (designed; in flight on PR #10) The `/healthz` 503 branch (unexercised as of CARD-001, per its review) is pinned two ways: at the HTTP layer, `server_test.go` injects a failing `Deps.Health` and asserts 503 plus a nil-returning one asserting 200; at the store layer, `store.Readiness` is tested with a real closed `*store.DB` (ping fails) and a non-existent `DATA_DIR` (`os.CreateTemp` fails) â€” both deterministic across uid, no mocks. Use a non-existent dir, **not `chmod 0500`**, for the not-writable case: `chmod` does not block root and is flaky in CI containers.
- [CARD-008] (designed; in flight on PR #11) Session and OAuth-`state` tokens and the OAuth `code`/`access_token` are secrets. CARD-001's request-log line logs `r.URL.Path` (not `RawQuery`/`RequestURI`), so the `code`/`state` in `/auth/callback?code=&state=` never reach the log â€” this card must not widen the log line to include the query, and `internal/auth` must never log a token, secret, or claim payload. Cookies live in the `Cookie` header, also unlogged.
- [kanban] Two design PRs opened in the same pump both append rows to `docs/adrs/README.md`, and the append point is the end of the same table â€” so each is `MERGEABLE` against `main` individually while **conflicting with each other** (confirmed via `git merge-tree`). GitHub's per-PR `mergeable` flag will not show this. Whichever design PR merges second needs a trivial rebase keeping both row-sets in ascending ADR order. Expect this whenever `/kanban` runs two ADR-bearing cards concurrently.

# Knowledge

Cross-card knowledge captured by `/kanban` from phase agents. Entries are prefixed
`[CARD-NNN]`. Decisions live in ADRs (`adr_dir`), not here.

## Conventions
- [CARD-001] GitHub Actions workflows should declare an explicit top-level `permissions:` block (least privilege, e.g. `contents: read`) rather than inheriting the repo-default `GITHUB_TOKEN` scope; add per-job write scopes only where a push/publish step needs them (the deferred registry-push job in `ci.yml` will).
- [CARD-001] Card scopes for this project are designed to map 1:1 onto the slices in `docs/implementation/06-vertical-slices.md`; card-slicer should treat that doc as strong evidence for the right-sizing verdict on infra/foundation cards, since the project's own architects already vertical-sliced the build order there.

- [CARD-001] Config is parsed by `config.Load(getenv func(string) string) (*Config, error)`, not by reading `os.Getenv` inside the package. `main` injects `os.Getenv`; tests inject a `map`-backed getenv. Keeps config parsing a pure, table-testable function with no process-env mutation.
- [CARD-001] Frontend assets are embedded through a single `web` package (`web/embed.go` with `//go:embed`), imported as `nyx-drop/web`. Go embed cannot reach parent directories, so the directive must live in the `web/` tree, not in `internal/server`. Later cards add files to the embed set here.
- [CARD-001] `server.New(server.Deps{...}) (http.Handler, error)` takes a `Deps` struct, not a positional arg list, so later cards add wiring (store, locks, auth) by adding fields without breaking the signature. Health readiness is an injected `func(context.Context) error` (S1 stub returns nil; CARD-002 supplies the real DB-ping + data-dir-writability check).

- [CARD-001] Site-host detection (`siteLabel`) is host-confusion-safe by construction: it matches on the full-dot suffix `"." + baseDomain` (never a bare `HasSuffix` on `baseDomain`), then rejects an empty label and any label still containing a dot. This defeats the classic suffix-confusion vectors (`evilsites.nyxhub.net`, `xsites.nyxhub.net`, `sites.nyxhub.net.evil.com`, `.sites.nyxhub.net`) — all resolve to a branded 404, none are treated as a site. CARD-003 must preserve this exact shape when it replaces the site-host stub with real serving, and must decide trailing-dot (FQDN) handling explicitly, since `normalizeHost` does not strip it (today it fails safe to 404).

## Gotchas
- [CARD-001] Invariant 9 (handlers use the injected Clock, never `time.Now()`) governs BUSINESS time (expiry). Request-latency measurement in the logging/metrics middleware must use `time.Now()`/`time.Since` (the monotonic wall clock) — a frozen fake Clock would report `dur_ms=0` and is semantically wrong for latency. Do not route latency timing through `clock.Clock`.
- [CARD-001] A single-pass "strip trailing `:port` if candidate is non-empty and colon-free" rule is idempotent on its own output, but composing it behind a bracket-unwrap step is NOT automatically idempotent — bracket content (e.g. `0:0` from `[0:0]`) can itself match the bare `host:port` shape and get re-stripped on a second call. Fix: route bracket content through the exact same idempotent single-pass helper before returning, rather than returning it raw. Any future host/address normalizer with an idempotency invariant should route every extraction branch through one shared, provably-idempotent primitive rather than duplicating similar-but-not-identical logic per branch — and back the invariant with live fuzzing (`go test -fuzz`, 30-60s), not just the seed corpus, before trusting it.
- [CARD-001] `.gitignore` entries for build artifacts that share a name with a real source directory must be anchored to repo root (e.g. `/drop`, not `drop`) — an unanchored pattern matches any path component with that name anywhere in the tree (here, `cmd/drop/` silently vanished from `git status`/`git add`). Worth a quick `git ls-files` sanity check after any `.gitignore` edit that adds a short/common name.

- [CARD-001] The `coverage_target` ("90% on the core logic layer") is an **aggregate over the layer's statements**, not a per-function floor. Measure it as `go test -coverprofile=… ./...` then sum covered/total statements across the named packages and functions — do not fail an individual function that sits below the number. A per-function reading blocked CARD-001 on `renderNotFound` at 71.4% while the layer stood at 92.0%, and the only way to satisfy it would have been to inject a breakable `fs.FS` into a compile-time `embed.FS` parse — deviating from a merged design's fixed signature to move a metric. Unreachable error paths are not coverage debt. CARD-012 lifts this rule into `PROTOCOL-ADDENDUM.md`.

- [CARD-001] `/healthz` and `/metrics` are matched in `top()` before host routing (invariant 8), so on a wildcard ingress **every site subdomain will serve unauthenticated Prometheus runtime metrics on any Host** unless the ingress excludes those paths. Not a defect — the spec mandates host-independent `/metrics`, and the exposition carries no secrets — but **CARD-011's Helm/ingress work must decide** whether `/metrics` is reachable from the public internet or restricted to in-cluster scraping / a separate port.
- [CARD-001] `internal/config/size.go`'s `ParseSize` accepts bare single-letter `K`/`M`/`G` decimal aliases (see `sizeSuffixes`) in addition to `KB/MB/GB` and `KiB/MiB/GiB`. This is visible on the unexported `sizeSuffixes` var but missing from `ParseSize`'s own doc comment and from `README.md`'s config table; keep all three in sync when ADR-0001's suffix set changes.
- [CARD-001] A multiply-overflow guard of the form `n > math.MaxInt64/mult` cannot be exercised by an input whose *digits alone* exceed int64 — `strconv.ParseInt` rejects those first. To test such a guard you need a value that fits int64 on its own but overflows after the suffix multiply (e.g. `9223372036854776kb`). A no-panic fuzz target will not catch a broken guard either. Check guard coverage with `go tool cover -func`, not by eyeballing the table.
- [CARD-001] `pr-review` lens agents running in parallel can collide on the `gh` review-POST round trip: one agent's review payload was silently submitted under another concurrent lens's content (detected by checksumming the request body and independently re-fetching the created review by id). The `[simplicity]` review landed twice on PR #3 as a result. Mitigation: write the review JSON to a lens-plus-random-suffix-named file, verify its checksum immediately before and after posting, and re-fetch the created review by id before trusting the POST response. Future `pr-expert-reviewer` dispatches should build this verify-after-post step in.
- [CARD-001] Go's `http.Server.IdleTimeout` falls back to `ReadTimeout` (**not** `ReadHeaderTimeout`) when zero, so setting only `ReadHeaderTimeout` leaves idle keep-alive connections unreaped — a resource-exhaustion vector. Always set `IdleTimeout` explicitly. Be cautious adding `WriteTimeout`/`ReadTimeout` to the process `http.Server` before the upload/site-serving cards land: a fixed deadline sized without knowing `MAX_UPLOAD_SIZE`/`MAX_SITE_SIZE` risks truncating legitimate large requests and responses. **CARD-003/004 must revisit `WriteTimeout`** once those limits are wired into the server.
